Jump Start for the Linux Realm Kit

Red Hat Linux uses an automated install system known as Kickstart. Very basically, this is a detailed text file that the Red Hat Linux installer can retreive and use to install the machine rather than asking the a human many questions. One of the most powerfull methods of delivering a kickstart file to the installer is by HTTP which allows complex scripting to generate the proper kickstart file. This is what we are doing here.

The Solaris Kit on campus uses the Solaris equivalent known as Jump Start which has a few key advantages. First, its well known and understood around campus, it uses include files much more easily that kickstart can, and most common configurations can be done in four or five lines rather than hundreds. Hence, this system was developed to take a config file in the form of a Solaris Jump Start, convert it to a Red Hat Kickstart, and HTTP it directly to the requesting client for instalation.

Maintained by Jack Neely.

Documentation

This system is desinged to generate a Red Hat Linux Kickstart to install the NCSU Realm Kit for Red Hat Linux. A config file can be used to customize the kickstart that is delivered. There are two ways to produce a config file. A normal kickstart file can be used or a Jump Start style config file can also be used. The files must be named the same as the target computers hostname. Files must be placed under the following directory:

/ncsu/jjneely/web-kickstart/configs.
